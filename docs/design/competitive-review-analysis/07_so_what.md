# So-What Summary

The ecosystem being analyzed is the full spectrum of code review excellence: AI-powered PR review tools (CodeRabbit, PR-Agent/Qodo, Greptile), human review methodology (Google Eng Practices, Chromium), academic research (EASE 2026, PatchGuru), and community prompting techniques - all evaluated for what wr.mr-review can adopt to improve finding quality and codebase-specificity. The near-term threat is not CodeRabbit (which is an infrastructure-heavy platform at $48/dev/mo pursuing a different buyer) but the PR-Agent open-source community, which is shipping MCP integration, AGENTS.md codebase context, and per-repo review rules within the next 6-12 months - capabilities that narrow wr.mr-review's differentiation if our reviewer families remain generic and shallow (confidence: medium, hedged by community velocity uncertainty). The single switching driver that matters most is signal-to-noise discipline: the #1 confirmed reason AI review tools get disabled is producing too many low-confidence findings, and every tool in the ecosystem - from PR-Agent's 3-findings default to CodeRabbit's self-reflection scoring - is explicitly designed around this constraint, while wr.mr-review currently has no equivalent volume cap. The recommended decision, traceable to the three "urgent" items in the decision ledger, is: before any architectural investment in codebase exploration or cross-repo analysis, ship the three low-complexity prompt-level improvements - two-pass self-reflection scoring, confidence constraint language in reviewer family prompts, and per-family finding caps - because these directly address the primary adoption gate and can be done within the existing session architecture without infrastructure changes. The single piece of new information that would most change this read is whether developers in high-risk engineering segments (fintech, healthcare, security-critical teams) recognize "review methodology enforcement" as a job they need done - if they don't, the positioning story must pivot from "we enforce methodology" to "we produce the highest-confidence findings," which would change which features to build first.
